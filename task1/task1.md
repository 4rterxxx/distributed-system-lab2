# Задание 1: Написать параллельную программу с использованием OpenMP для нахождения множества Мандельброта. 
 


### 1. Постановка задачи  

**Цель работы:** Реализация параллельной программы для вычисления точек, принадлежащих множеству Мандельброта, с использованием технологии OpenMP.  

**Математическая формулировка:** Множество Мандельброта — это набор комплексных чисел c, для которых последовательность zₙ₊₁ = zₙ² + c (при z₀ = 0) остается ограниченной. Практический критерий: |zₙ| < 2 для всех n.  

### 2. Алгоритм решения  

Для каждой точки c в заданной области комплексной плоскости:  
1. Задаем начальное значение z = 0 + 0i  
2. Выполняем до MAX_ITER раз:  
   - Вычисляем z = z² + c  
   - Если |z| ≥ 2 → точка не принадлежит множеству (выходим из цикла)  
3. Если выполнили все итерации без выхода → точка принадлежит множеству

### 3. Реализация  

#### Структура программы  

```c
Основные компоненты:
1. Функция проверки принадлежности точки множеству
2. Создание равномерной сетки точек
3. Параллельный цикл по строкам сетки
4. Сбор результатов из всех потоков
5. Запись в CSV-файл
```

### 7. Выводы

1. **Линейное масштабирование для больших задач:**
   - Для 1M точек: 4 потока дают ускорение 4.06x (101.5% эффективности)
   - Это близко к идеальному масштабированию

2. **Оптимальное количество потоков зависит от размера задачи:**
   - 10k точек: оптимально 2-4 потока
   - 100k точек: оптимально 4 потока
   - 1M точек: оптимально 4-8 потоков

3. **Эффективность снижается при малых задачах:**
   - Чем меньше задача, тем выше накладные расходы на параллелизацию


Программа успешно вычисляет множество Мандельброта с использованием OpenMP. Для 1 миллиона точек достигнуто линейное ускорение: 4 потока дают ускорение 4.06x, 8 потоков — 5.84x. Эффективность растёт с увеличением размера задачи: малые объёмы невыгодно распараллеливать на много потоков из-за накладных расходов, тогда как для больших задач параллелизация эффективна. Оптимально использовать 4 потока для 100k точек и 4-8 потоков для 1M точек.
